El problema

La paradoja de la elección

Demasiadas opciones no liberan al usuario — lo paralizan. En el ecosistema actual de plataformas digitales, las reseñas de desconocidos generan sobrecarga de información. Sin embargo, las decisiones de consumo siguen profundamente ligadas a la confianza: lo que recomienda un amigo pesa más que mil reseñas anónimas.

El objetivo fue construir un sistema de recomendación personalizado que transforme el dataset masivo de Yelp en sugerencias con significado — incorporando no solo patrones de comportamiento individuales, sino también la influencia de la red social del usuario.

Por qué las 5 estrellas no alcanzan

Los sistemas de rating tradicionales tienen un problema estructural: el sesgo positivo. El promedio de ratings en el dataset es 3.93 sobre 5, y la distribución está fuertemente sesgada hacia calificaciones altas. Los usuarios escriben reseñas principalmente luego de experiencias excepcionales, no promedio.

Además, cada usuario tiene una escala diferente: el usuario 52500 tiene un promedio de 4.60, mientras el usuario 51007 promedia 3.59. Una estrella de uno no equivale a una estrella del otro. El sistema de recomendación debe capturar estas diferencias individuales.

Los datos

Dataset Yelp

El dataset original de Yelp contiene 1.84 millones de reseñas con 40 variables: datos del usuario (id, amigos, tipo de usuario, índice de influencia), del restaurante (nombre, ubicación, cocina principal, atributos como WiFi, alcohol, reservas) y de la reseña (rating, texto, fecha, score de impacto). Para el modelado se filtró el subconjunto de Restaurantes ubicados en Philadelphia.

El dataset filtrado tiene 8.714 usuarios únicos y 3.372 restaurantes únicos, con 111.831 ratings reales sobre un universo teórico de 29.3 millones de combinaciones posibles. Esa alta dispersión — menos del 0.4% de la matriz completa — es precisamente el problema que el sistema de recomendación viene a resolver. El grafo social construido con NetworkX conectó esos usuarios con 32.917 vínculos entre amigos.

Los modelos

Cinco enfoques progresivos

Cada modelo agrega una capa de inteligencia sobre el anterior, desde el baseline más simple hasta la integración de texto y red social.

M1
Rank Based (baseline)
Recomienda los restaurantes con mejor rating promedio (mín. 55 interacciones). Útil para el cold start: usuarios nuevos sin historial. Sin personalización.
M2
Collaborative Filtering — User-User (KNN)
Identifica usuarios con patrones de rating similares. Recomendaciones basadas puramente en comportamiento, sin metadata del restaurante. Recall 82%, F1 0.777.
M3
Collaborative Filtering — Matrix Factorization (SVD)
Descompone la matriz usuario-restaurante en factores latentes. Menor RMSE que KNN (1.03 vs 1.12). Precisión 72%, Recall 79%.
M4
SVD + Social Graphs (Friends)
Incorpora la red social del usuario al SVD mediante regularización social: las valoraciones de los amigos "guían" la predicción. Especialmente útil para restaurantes con datos escasos.
M5
SVD + Sentiment Analysis (VADER) elegido
Enriquece el SVD con el score de sentimiento extraído del texto de cada reseña usando VADER. El sentimiento del texto correlaciona positivamente con el rating — aporta señal que el número de estrellas solo no captura. Recall 84%, F1 0.783.

Recomendación

Estrategia híbrida por sección

El sistema no propone un único modelo sino una arquitectura por secciones dentro de la plataforma, según el estado del usuario:

Usuarios nuevos (cold start)
Rank Based — los restaurantes mejor valorados con suficiente volumen de reseñas.
"Basado en tus reseñas anteriores"
SVD + Sentiment Analysis — el modelo con mayor recall para usuarios activos.
"Lo que recomiendan tus amigos"
SVD + Friends — aprovecha la red social para descubrir lugares fuera del historial propio.
"Restaurantes similares a tu historial"
CF Item-Item — encuentra locales con perfiles parecidos a los que ya visitaste.
Conclusiones

Lo que aprendimos

1
El texto de la reseña contiene señal que el rating no captura. El análisis de sentimiento con VADER mostró correlación positiva clara entre texto y estrellas — integrar ambos mejoró el Recall del modelo base.
2
La red social amplifica la recomendación, no la reemplaza. El modelo de amigos fue especialmente valioso para restaurantes con pocos datos (long tail), donde el filtrado colaborativo clásico falla.
3
RMSE no es la métrica correcta para este problema. Predecir el rating exacto importa menos que capturar todos los lugares que el usuario disfrutaría. Recall fue la guía real del desarrollo.
4
Un solo modelo no sirve para todos los usuarios. La arquitectura híbrida por sección resuelve el cold start y maximiza la personalización según el ciclo de vida del usuario en la plataforma.
